# Phase 2 说明书提取 - 暂停总结

**日期**: 2026-05-11  
**决策**: Phase 2 暂停，专注 Phase 1

---

## 调试过程

### 问题 1: API 权限 403 错误

**现象**: 
- 使用 `/open-apis/drive/v1/medias/{file_token}/download` 返回 403
- 多维表格附件字段无法下载

**排查步骤**:
1. 检查飞书应用权限配置
2. 添加 `drive:file:download` 权限
3. 测试不同 API 端点

**解决方案**: ✅
- 正确 API: `/open-apis/drive/v1/files/{file_token}/download`
- 云盘文件可以下载（多维表格附件不行）

### 问题 2: PDF 内容提取失败

**现象**:
- 成功下载 47.58 MB PDF 文件
- pypdf 提取到 0 字符文本

**根本原因**: ❌
- 说明书是**图片型 PDF**（扫描版）
- 无法用 pypdf 直接提取文本
- 需要 OCR 识别

---

## 暂停原因

1. **技术限制**: 图片 PDF 需要 OCR，pypdf 无法提取
2. **成本考虑**: 682 个文件 × OCR 识别 = 时间长、成本高
3. **准确率**: OCR 准确率不确定，尤其多语言说明书
4. **优先级**: Phase 1 已完成，可先上线使用

## 技术发现

### 飞书云空间 API

**文件下载端点区别**:
```
❌ /drive/v1/medias/{file_token}/download
   - 用途: 媒体文件（图片、视频）
   - 多维表格附件: 不支持
   
✅ /drive/v1/files/{file_token}/download
   - 用途: 普通文件（PDF、Word、Excel）
   - 云盘文件: 支持 ✅
   - 多维表格附件: 不支持 ❌
```

**权限要求**:
- `drive:file:download` (免审权限)
- 需要在飞书后台手动勾选并发布

### 多维表格附件限制

多维表格的附件字段（Attachment type 15）:
- 存储格式: `{"link": "...", "text": "filename"}`
- 无法通过标准 Drive API 下载
- 可能需要专用 API 或导出功能
