## 1. 修复 @提及 响应问题

### 问题描述
机器人无法响应飞书群中的 @提及 查询，日志显示：
```
INFO - Ignored message type: post
INFO - Received empty message
```

### 根本原因
- 机器人只处理 `text` 类型消息
- @提及发送的是 `post` 类型（富文本消息）
- post 消息解析逻辑错误，导致提取出空文本

### 解决方案

**修改文件：** `bot/main.py`

1. **支持 post 消息类型**
   ```python
   if message_type in ('text', 'post'):  # 原来只有 'text'
   ```

2. **正确解析 post 消息结构**
   ```python
   elif message_type == 'post':
       # 飞书post消息的实际结构：
       # {"content": [[{"tag": "text", "text": "消息内容"}]]}
       if 'content' in content_json:
           content_blocks = content_json.get('content', [])
       else:
           # 兼容嵌套结构 post.zh_cn.content
           post_content = content_json.get('post', {})
           lang_content = post_content.get('zh_cn') or post_content.get('en_us') or {}
           content_blocks = lang_content.get('content', [])
   ```

3. **提取文本内容**
   ```python
   for paragraph in content_blocks:
       for element in paragraph:
           if element.get('tag') == 'text':
               text_parts.append(element.get('text', ''))
   ```

### 测试结果
✅ @提及查询正常响应  
✅ 普通文本消息不受影响

## 2. 性能优化：并行数据库查询

### 问题描述
机器人响应速度慢，单次查询耗时 ~15-16秒：
- 数据库查询：~6.5秒（串行）
- DeepSeek API：~5.5秒
- 日志记录：~2秒（阻塞主流程）

### 优化方案

#### 2.1 并行数据库查询

**修改文件：** `bot/search.py`

**原逻辑（串行）：**
```python
product_results = search_products_by_sku(sku)      # 等待2.5秒
knowledge_results = search_by_sku_exact(sku)       # 再等待2秒
results = product_results + knowledge_results
```

**新逻辑（并行）：**
```python
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=2) as executor:
    future_product = executor.submit(search_products_by_sku, sku)
    future_knowledge = executor.submit(search_by_sku_exact, sku)
    
    # 收集结果（带超时和异常处理）
    for future, name in [(future_product, 'products'), (future_knowledge, 'knowledge_entries')]:
        try:
            result = future.result(timeout=10)  # 10秒超时
            if name == 'products':
                product_results = result or []
            else:
                knowledge_results = result or []
        except Exception as e:
            logger.warning(f"Search failed for {name}: {e}")
            # 失败的查询不影响另一个
```

**关键特性：**
- ✅ 两个查询完全独立，可以并行
- ✅ 任何一个失败不影响另一个
- ✅ 每个查询有独立的10秒超时保护

#### 2.2 异步日志记录

**修改文件：** `bot/handlers.py`

**原逻辑（阻塞）：**
```python
search_result = smart_search(query)
log_search(user_id, query, search_type, len(results))  # 阻塞2秒
```

**新逻辑（异步）：**
```python
search_result = smart_search(query)

# 后台线程记录日志，不阻塞主流程
import threading
log_thread = threading.Thread(
    target=log_search,
    args=(user_id, query, search_type, len(results)),
    daemon=True
)
log_thread.start()
```

## 3. 敏感信息过滤

### 需求背景
知识库中可能包含采购价格、供应商等敏感信息，不应该返回给普通用户。

### 实施方案

**修改文件：** `bot/search.py`

**新增函数：** `filter_sensitive_content(results)`

```python
def filter_sensitive_content(results: List[Dict[str, Any]]) -> List[Dict[str, Any]]:
    """
    过滤包含敏感信息的记录（价格、供应商等）
    """
    # 敏感关键词列表
    SENSITIVE_KEYWORDS = [
        # 价格相关
        '价格', '采购价', '成本价', '进价', '售价', '报价', '单价', '总价',
        '成本', '费用', '金额', '人民币', '美元', '雷亚尔',
        'USD', 'BRL', 'CNY', 'R$', '$', '¥', '元',
        # 供应商相关
        '供应商', '供货商', '厂家', '生产商', '工厂', '制造商',
        'supplier', 'vendor', 'factory', 'manufacturer'
    ]

    filtered_results = []
    filtered_count = 0

    for result in results:
        title = result.get('title', '').lower()
        content = result.get('content', '').lower()
        combined_text = title + ' ' + content

        # 检查是否包含敏感关键词
        is_sensitive = any(keyword.lower() in combined_text for keyword in SENSITIVE_KEYWORDS)

        if not is_sensitive:
            filtered_results.append(result)
        else:
            filtered_count += 1
            logger.info(f"Filtered sensitive content: SKU={result.get('sku')}")

    if filtered_count > 0:
        logger.info(f"Filtered {filtered_count} sensitive records")

    return filtered_results
```

**集成到搜索流程：**
```python
def smart_search(query: str, limit: int = 10):
    # ... 执行查询 ...
    results = product_results + knowledge_results
    
    # 过滤敏感信息
    filtered_results = filter_sensitive_content(results)
    
    return {
        'results': filtered_results,
        'total_before_filter': len(results),
        'filtered_count': len(results) - len(filtered_results)
    }
```

### 过滤规则

**被屏蔽的内容：**
1. **价格信息** - 任何包含价格、金额、货币符号的记录
2. **供应商信息** - 任何包含供应商、厂家等关键词的记录

**日志示例：**
```
INFO - Filtered sensitive content: SKU=ABC123, title=XXX产品采购价格...
INFO - Filtered 3 sensitive records out of 10 total results
```
